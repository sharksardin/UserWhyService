뷰티 상품 글로벌 진출을 돕기 위해 상품명 입력 시 AI 가 해외 시장 적합도를 평가하고 최적의 국가를 추천하는 웹 대시보드 입니다.

1. 설계의도

   이 프로젝트는 AI와의 빠르고 안정적인 통신과 제한 시간 내 핵심 비즈니스 가치 증명에 초점을 맞추었습니다.

   Backend (FastAPI) : Pydantic을 활용해 프론트엔드와 주고받는 JSON 스키마를 여러 테스트로 검증하여 AI의 환각으로 인한 데이터 포맷 붕괴를 최소화 하였습니다.
   Frontend (Next.js + Tailwind CSS) : 실제 유저와이의 프리셀링 진단 리포트 UI를 분석하여 사용자가 즉각적인 의사결정을 내릴 수 있도록 상태 라벨 기반의 카드형 대시보드로 설계하였습니다.
   Infrastructure (Docker Compose) : Nginx를 활용한 리버스 프록시 도입도 고려하였으나 과제 권장 시간 대비 오버 엔지니어링이라 판단했습니다. 대신 FastAPI와 Next.js의 자체서버를 활용한 Docker Compose 구조를 채택하여, 환경의 이식성을 확보하면서 비즈니스 로직 개발에 집중하는 방식을 선택했습니다.

2. 테스트 전략

   단위테스트의 양보다 의도와 품질을 중요시하여 AI-Native 서비스의 핵심인 LLM응답의 신뢰성 및 제약조건 검증에 집중했습니다.

   예외처리 검증: 빈 문자열 입력 시 시스템이 다운되지 않고 400 Bad Request를 올바르게 반환하는 지 검증했습니다.
   프롬프트 제약 조건 검증: AI가 프롬프트 지시를 무시하지 않는지 다음을 검증하였습니다.
   1. 완벽한 JSON 포맷을 반환하는지,
   2. 'total_score'가 0~100 사이의 정수인지,
   3. 'status_label이 지정된 Enum 값(GO/CAUTION/STOP) 내에서만 도출되는지 코드로 검증했습니다.

3. AI 도구 활용 보고서

   본 프로젝트는 기획부터 인프라 배포까지 AI 어시스턴트와 페어 프로그래밍을 진행했습니다.
   활용 영역: FastAPI 라우터 보일러 플레이트 작성, Pydantic 스키마 뼈대 구성, Next.js 기반의 프론트엔드 UI 컴포넌트 스캐폴딩 등 초기 세팅 시간을 획기적으로 단축하는 데 사용했습니다.
   수용 및 거절/수정의 판단 기준: (수용): AI가 제안한 기본적인 UI 레이아웃과 디자인 컬러 시스템은 완성도가 높아 수용했습니다.
   (수정): AI가 제안한 기본 'docker-compose' 빌드 시 파이썬 가상환경('venv')과 Node.js 모듈('node_modules')까지 Build Context 에 포함되어 용량이 800MB이상 폭증하는 문제가 있었습니다. 이를 직접 인지하고 '.dockerignore'를 각각 구성하여 빌드 용량을 약 150MB 감소시켰습니다.
   효과와 한계: AI는 단순 스캐폴딩과 프로토타이핑에서 압도적인 생산성을 제공합니다. 하지만 외부 API 연동 시의 Edge case 대처나, 오라클 클라우드 환경의 OS 레벨 방화벽 제어 및 Docker 빌드 최적화 같은 인프라 이슈는 여전히 엔지니어의 비판적 검증과 아키텍처 통제력이 필수적임을 깨달았습니다.
